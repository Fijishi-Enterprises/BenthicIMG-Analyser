Coral Point Count (CPCe) saves point positions and annotations in .cpc files - one .cpc file per image. You can upload .cpc files here to import the points and annotations into your source.

You can select multiple files, but you can only select files within a single folder. Tip: When selecting files, click inside the folder area and then hit Ctrl+A in Windows / Cmd+A in Mac to select all files in that folder.

  • The image filenames used by the CPC files must match existing image filenames in your CoralNet source. So if your CPC file looks for DSC_0048.JPG, then the image name on CoralNet should be DSC_0048.JPG as well. This is how CoralNet figures out which CPC file belongs to which image. The name of the folder containing your image file does not matter.
  • When you upload a CPC file for an image, that image's existing points and annotations on CoralNet will be overwritten. After you select your CPC files, you'll see a preview that will warn you if you're about to overwrite confirmed annotations.
  • Each CPC file is allowed to contain labels for all, some, or none of the points.
  • Label short codes used in the CPC file must match a label code in your source's labelset.
    • If you haven't created a labelset for your source yet, you can do so here.
    • Although labels are site-wide in CoralNet, you can customize the labels' short codes for your source. To do this, go to your source's "Labelset" page, then from there go to the "Edit label entries" page.
    • Codes are case-insensitive. If your labelset has a label code PORIT, and the CPC file has Porit, then these codes will match.

"Determine CoralNet label codes from" option: CPCe's interface has two fields for each point, "ID" and "Notes". Normally, we just take the ID as the label code to save in CoralNet. However, if you choose to determine label codes from "ID and Notes fields", we instead combine the ID and Notes codes to form the label code for CoralNet. For example, if the ID field has "ROCK" and the Notes field has "TURF", then the CoralNet label code is taken to be "ROCK+TURF". Note that this only works if your source's labelset has an entry with the short code "ROCK+TURF". Also keep in mind that CoralNet short codes can't be more than 10 characters long.

CPC files contain other information besides point locations, ID, and Notes - such as header data. When you upload CPC files, CoralNet retains this information on a per-image basis, and you can get this information back by exporting annotations to CPC format. (The Export feature is available at the bottom of the "Images" page.)

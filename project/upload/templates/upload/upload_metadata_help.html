Your CSV (comma-separated values) file should have one row per image, and one column per metadata field that you want to specify. The first row must contain column headers that specify which column is which. Here's an example:

Name Date {{ aux_field_labels.0 }} Camera
IMG_0056.jpg 2015-04-17 Value A Nikon
DSC_0162__postprocess.png 2015-04-28 Value C

Recognized column names:

    {% for field_label in field_labels %}
  • {{ field_label }}
    • {% endfor %}

You can use a spreadsheet program such as Excel to create the CSV file. It's not much different from creating a normal spreadsheet; just select the CSV format in the "Save As" dialog. Alternatively, depending on how your metadata is organized, you could save some manual work and automate the CSV file creation with a programming language like Python.

Details on how your CSV file will be processed:

  • If a non-blank metadata value is going to be overwritten, that value will be shown alongside the CSV's value in the preview.
  • Blank CSV cells are accepted. You can also use blank cells to overwrite non-blank metadata values.
  • The Name column is required. You can specify as many or as few of the other columns as you like. If a column is excluded, that metadata field will be untouched.
  • You may include rows with image names that aren't in your source. Those rows will just be ignored.
  • You may include extra columns with names outside of the recognized column names. Those columns will just be ignored.
